You are a senior full-stack AI engineer and system architect.

Your task is to build a complete working prototype called **Research Radar AI** based strictly on the following PRD and TRD requirements.

The goal is to create a **live research intelligence dashboard** that analyzes emerging technologies using real-time data from research papers and GitHub repositories.

The system must be optimized for a **hackathon demo environment running on Replit**.

---

# PRODUCT OVERVIEW

Research Radar AI is a web platform that:

• Fetches latest research papers from arXiv
• Fetches trending GitHub repositories
• Extracts keywords using NLP
• Detects technology trends
• Displays insights in a dashboard

The system must support **live queries from judges during a demo**.

Example:

User types:

"quantum AI chips"

System must:

1. Fetch latest arXiv papers
2. Fetch related GitHub repositories
3. Extract keywords
4. Compute a simple trend score
5. Display results in an interactive dashboard

The system must show **timestamps and source links** so judges can see it is not static data.

# BACKEND REQUIREMENTS

Use FastAPI.

Implement the following endpoints:

GET /api/health

GET /api/arxiv?q=topic

GET /api/github?q=topic

GET /api/analyze?q=topic

The analyze endpoint must:

1. Fetch arXiv papers
2. Fetch GitHub repos
3. Extract keywords from abstracts
4. Compute a trend score
5. Return structured JSON response

Example response:

{
"query": "quantum ai chips",
"arxiv_count": 6,
"github_count": 4,
"keywords": ["neuromorphic", "quantum", "accelerator"],
"trend_score": 0.74,
"items": {
"arxiv": [...],
"github": [...]
}
}

Use:

feedparser for arXiv
requests for GitHub
scikit-learn TF-IDF for keyword extraction.

Add caching using SQLite.
