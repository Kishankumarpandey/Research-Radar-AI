Project Overview

Name: Research Radar AI
संक्षेप: एक वेब-आधारित AI प्लेटफ़ॉर्म जो arXiv, GitHub, patent feeds और tech news को real-time/near-real-time में स्क्रैप करके NLP से analyze करता है और “technology radar / trend” दिखाई देता है — startup opportunities और research surge alerts के साथ। Demo Replit पर चलेगा और judges के सामने live API calls दिखा सकेगा।

Product Vision

Mission: कंपनियों, रिसर्चर्स और छात्रों को पहला-हाथ, actionable insight देना — कौन-सी तकनीकें उठ रहीं हैं और कहाँ पर opportunity/competition कम है।
Value: कम समय में high-signal research trends, startup leads और country-wise innovation map मिलें। Hackathon के लिए emphasis: interactive live demo — judges किसी भी topic डालें, system live fetch + analyze करके दिखाए।

Key User Flows

Flow A — Judge Live Demo (critical)

Open Dashboard → Click “Live Analyze”

Input topic: quantum AI chips

System makes arXiv API call + GitHub search (live)

Show progress spinner → display: (a) top 5 latest papers with abstracts (b) extracted keywords (c) radar/trend update

Judge sees timestamps & source links → impressed by live behavior

Flow B — Researcher Exploring Tech

Login (optional) → Enter topic & time window

Save analysis → schedule repeating weekly scan (future feature)

Flow C — Browse Trending Technologies

Open “Global Radar” → See ranked list + country heatmap → click a tech to view details

Success Metrics (how judge / team will measure demo success)

Demo impress criteria: Live Fetch Visible (Judge can see network calls & timestamps) — qualitative.

Quantitative (for product):

% of queries returning ≥5 relevant papers within 10s (target MVP: 80%)

Accuracy of top-3 keywords judged relevant by user (manual eval)

Dashboard latency for live query (MVP target: <15s for small N)

Hackathon success signals: judge’s “system is dynamic” comment + working prototype link.

Out of Scope (MVP)

Full user auth / multi-tenant infra (can be added later)

Paid data integrations (Scopus / Web of Science) — not in MVP

Production-grade ML training pipeline (MVP uses lightweight topic extraction)

Any heavy GPU-based model inference on Replit (avoid unless using small models)

Development Phases & Deliverables (phases, not dates)

Phase 0 — Setup & repo skeleton

Create GitHub repo, Replit project.

Basic README, environment variable template (.env.example).

Phase 1 — Data Fetch & API (MVP core)

Implement GET /api/arxiv?q=<topic>&max=10 (FastAPI)

Implement GET /api/github?q=<topic>&max=10 (server side, with token)

Basic caching layer (in-memory / SQLite) to avoid rate limits

Phase 2 — Analysis

Keyword extraction (TF-IDF) + small sentence embeddings for dedup

Trend score (count results for last 6/12 months, compute growth%)

Phase 3 — Frontend Dashboard

Next.js / React single-page app

Components: LiveSearch, PapersList, RadarChart, GitHubList, ProgressIndicator

Phase 4 — Demo polish

Add "Run Live" progress UI, timestamps, source links

Add methodology modal (explain metrics)

Prepare demo script & sample queries

Phase 5 — Optional enhancements

BERTopic clustering, country mapping, alerts

Privacy & Safety / Legal

Data licensing: Use public APIs (arXiv, GitHub) and follow their Terms of Service. Always link back to source.

No PII collection in MVP. If adding user accounts later, follow secure password storage and privacy policy.

Rate limits & caching: Implement caching & user-visible timestamps to show when data was fetched.

Attribution: Show source (arXiv / GitHub URL) for each item.

Safety: Don’t provide fabrications — if low results, show “no strong signal found”.

Definition of Done (DoD checklist)

 GET /api/arxiv?q= returns live results from arXiv with titles, abstracts, date, link.

 GET /api/github?q= returns top repos (name, stars, url).

 Frontend LiveSearch triggers backend calls and displays results with timestamps.

 Radar chart updates after every live run.

 Demo script exists (3 sample topics + expected outputs).

 README contains Replit deploy instructions and env var list.

 Basic caching implemented to prevent spamming APIs.

 Methodology modal explains how trend score is calculated.

 No PII collected; all sources attributed.

Design System (brief)

Colors / UI: Clean, high-contrast dashboard. Suggested palette — dark theme for demo screens (judges like crisp dark UIs).
Components: Top nav, search input with suggestion, progress spinner, cards for papers/repo, radar chart (Chart.js), heatmap (D3.js optional).
Typography: Sans-serif, clear sizes for headings / paper titles.
Accessibility: links open in new tabs; alt text for icons.

Replit-Specific Implementation Notes (practical steps)

Recommended stack on Replit (simple & deployable):

Backend: FastAPI (Python) — easy to run on Replit.

Frontend: React (Vite) or simple React app served as static. You can also use a single Replit to run both (serve frontend static files from backend) or use two repls.
