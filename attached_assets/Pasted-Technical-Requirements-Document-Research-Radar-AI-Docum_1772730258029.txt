System Architecture

High level components

Frontend (React / Vite / Next.js) — User input, live run control, charts.

Backend (FastAPI) — API endpoints, data fetchers, analysis engine, caching.

Data Fetchers — arXiv client (feedparser/requests), GitHub client (tokened requests).

Analysis Engine — TF-IDF / sentence-embeddings + simple trend scoring. (Optional: BERTopic)

Cache / DB — SQLite (Replit friendly) or Replit DB / in-memory cache (LRU).

Static hosting / Deployment — Replit (single repl serving FastAPI + static) or split Frontend (Vercel) + Backend (Replit).

Monitoring / Logs — simple log files + health endpoint.

Flow:
User → Frontend → /api/analyze?q= → Backend: check cache → fetch arXiv & GitHub → analyze → store short cache → return results → Frontend updates charts.

CREATE TABLE query_cache (
  id INTEGER PRIMARY KEY AUTOINCREMENT,

  payload JSON NOT NULL,

API Design
Authentication for admin endpoints

Admin / secrets endpoints protected by token in env (API_KEY).

Public endpoints (MVP)

GET /api/health

Response: { "status": "ok", "time": "..." }

GET /api/arxiv?q=<topic>&max=10

Live fetch from arXiv (or return cached).

Response: { source: "arXiv", query, count, items: [{id,title,summary,published,link}] , fetched_at }

GET /api/github?q=<topic>&max=10

Uses GITHUB_TOKEN from env. Rate limit handling.

Response: { source:"GitHub", query, count, items: [{name, html_url, stars, description}] }

GET /api/analyze?q=<topic>&max=10

Orchestrates arXiv + GitHub fetch + analysis.

Response:

GET /api/health/runlog?last=10 (optional) — returns run logs.

Error handling

Standard HTTP codes (400, 401, 429, 500).

On external API failure: return partial result with errors field and fetched_at timestamps for each source.

Rate limiting & headers

Include response headers: X-Cache-Hit: true/false, X-RateLimit-Remaining (if available from GitHub).

Security & Rate Limiting

Secrets

GITHUB_TOKEN stored in Replit secrets or .env (never commit).

API admin key for protected endpoints.

Rate limiting

Per IP: limit requests to backend (e.g., 30/min) — use slowapi or simple in-memory token bucket.

External APIs: respect GitHub rate limits; implement exponential backoff and caching.

Cache per query TTL: 10 minutes default; configurable.

Input sanitization

Validate q parameter length (min 2, max 120), allow only safe chars.

Escape outputs in frontend to avoid XSS.

Privacy

No PII stored in MVP. If adding accounts later, follow standard bcrypt + JWT.

Provenance & Attribution

Always return source links (arXiv/GitHub) and fetched timestamps.

AI Integration (if applicable)

MVP

Keyword extraction: TF-IDF (sklearn) on fetched abstracts + title.

De-dup: sentence embeddings (SBERT small model like all-MiniLM-L6-v2) to cluster near-duplicates.

Trend score: simple heuristic — count of papers/repos in last 6 months vs previous 6 months → normalized growth metric (0–1).

v1 / advanced

Topic modeling: BERTopic (requires more memory).

Embedding store: FAISS (for fast similarity) — optional when scaling.

Prediction: a simple time series model (Prophet) to estimate growth — optional offline training.

Model hosting

Use small models that run on CPU for Replit. If heavy models needed, host embeddings/prediction on a remote GPU service or use sentence-transformers hosted embeddings API.

FastAPI arXiv endpoint (minimal)

# backend/app.py (snippet)
from fastapi import FastAPI, Query
import feedparser, time
from datetime import datetime, timedelta
import sqlite3, json

app = FastAPI()

def fetch_arxiv(q, max_results=10):
    url = f"http://export.arxiv.org/api/query?search_query=all:{q}&start=0&max_results={max_results}"
    feed = feedparser.parse(url)
    items = []
    for entry in feed.entries:
        items.append({
            "id": entry.id.split('/')[-1],
            "title": entry.title,
            "summary": entry.summary,
            "published": entry.published,
            "link": entry.link
        })
    return items

@app.get("/api/arxiv")
def arxiv_search(q: str = Query(...), max: int = 10):
    start = time.time()
    items = fetch_arxiv(q, max)
    duration = int((time.time()-start)*1000)
    return {"source":"arXiv","query":q,"count":len(items),"items":items,"fetched_at":datetime.utcnow().isoformat()+"Z","duration_ms":duration}

Simple TF-IDF keywords (Python)

from sklearn.feature_extraction.text import TfidfVectorizer
def extract_keywords(texts, top_k=10):
    vec = TfidfVectorizer(stop_words='english', ngram_range=(1,2), max_features=5000)
    X = vec.fit_transform(texts)
    sums = X.sum(axis=0)
    terms = vec.get_feature_names_out()
    term_weights = [(terms[i], sums[0,i]) for i in range(len(terms))]
    term_weights.sort(key=lambda x: x[1], reverse=True)
    return [t for t,w in term_weights[:top_k]]
